# MD-Manager

A simple markdown-based system to support engineering manager activities.

## Overview

This system uses markdown files to manage and track various engineering management tasks and documentation.

## Folder Structure

- **`/meetings/`** - Meeting notes organized by date or person
- **`/todos/`** - Task tracking and action items
- **`/people/`** - Team member profiles and 1-on-1 notes
- **`/projects/`** - Project-specific documentation
- **`/processes/`** - Team processes and workflows
- **`/decisions/`** - Decision records and ADRs
- **`/templates/`** - Reusable templates for consistency

## Quick Start

### Creating a new meeting note
Copy a template from `/templates/` to `/meetings/` and rename with the date and topic.

### Tracking tasks
Use `/todos/active-tasks.md` for day-to-day task management.

### Team member info
Create a file in `/people/` using the person-profile template.

## Templates Available

- `1-on-1.md` - One-on-one meeting template
- `team-meeting.md` - Team meeting template
- `person-profile.md` - Team member profile
- `decision-record.md` - Decision documentation
- `project-brief.md` - Project overview and tracking

## Getting Started

1. Review and customize the templates to fit your needs
2. Create your first team member profiles in `/people/`
3. Start tracking tasks in `/todos/active-tasks.md`
4. Schedule and document your next 1-on-1 using the template
